# Авито.Работа (`job`)

API для размещения, редактирования и снятия с публикации вакансии Авито Работа
**Авито API для бизнеса предоставляется согласно [Условиям
  использования](https://www.avito.ru/legal/pro_tools/public-api).**

---

## Общая информация

С помощью методов API из этого раздела вы можете публиковать и редактировать
объявления в категории Работа, включать и отключать уведомления на отклики по
вашим вакансиям, искать резюме и покупать контакты.

## Работа с API
### Действующие ограничения на каждый API метод
* Для работы с API необходимо [подключить](https://www.avito.ru/tariff/cpa/profile) Максимальную подписку в Работе, [подробно](https://support.avito.ru/sections/294) о тарифах.
* Базовое ограничение на число запросов - 10 rps
* Для изменения установленного лимита обратитесь в службу поддержки
* В случае превышения ограничения на количество запросов, вы получите ответ с кодом `429` и заголовками:
  * **X-RateLimit-Limit** - текущее установленное ограничение на число запросов в минуту
  * **X-RateLimit-Remaining** - доступно оставшиеся количество запросов в текущем временном окне

## Работа с откликами
### Работа с откликами через обратный вызов (webhook)

Преимущества Webhook'а (*Предпочтительный способ интеграции*): 
* Реакция на события в реальном времени, при каких-либо изменениях в отклике происходит вызов Webhook'а, обновление резюме, прохождение чат-бота и т.д.
* Интеграция веб-приложение с внешними службами или API. Вместо опроса API с использованием cron, веб-приложение может принимать уведомления через Webhook о наличии новых данных и активно реагировать на них.
* Возможность указывать Webhook для каждого приложения.
* Позволяет избежать лишних опросов и запросов к API с использованием cron, даже когда нет новых данных или изменений, требующих обработки.
* Использование Webhook позволяет избежать необходимости ждать определенного времени, как в случае с использованием cron.

##### Алгоритм интеграции:
1. Реализуйте на стороне вашего сервиса POST-метод, который:
   * на вход принимает тело запроса формата `{"applyId": "623850d1d3819d935dd02702"}`, где `applyId `- это идентификатор отклика.
   * поддерживает заголовок `X-Secret`, для проверки подлинности источника данных.
   * проверьте доступность метода, перед сохранением Webhook'а на него уйдет POST-запрос с пустым body, а в заголовке передадим `X-Secret`. В ответ ожидается код ответа 2\*\* в течение одной секунды.
2. [Авторизация](#ApiDocumentationBlock), получить токен временного доступа.
3. [Включите уведомления на отклик](#operation/applicationsWebhookPut), в качестве url укажите ссылку на ваш POST-метод, в качестве secret укажите строку максимальной длины 100 символов, которая будет передаваться в заголовке X-Secret при запросе к вашему сервису.
   * метод создания/изменения webhook, проверит доступность созданного в п.1 метода из контура Авито, отправив запрос с пустым телом по указанному url. 
4. [Получить список ваших вакансий](../item/documentation#operation/getItemsInfo), для фильтрации необходимо передать `category=111`, где 111 - это категория вакансии. Сохранить id вакансии и описание, что бы в дальнейшем понимать на какую вакансию произошел отклик.
5. [Получить информацию по откликам](#operation/applicationsGetByIds), передайте `applyId` в качестве параметра ids, не больше 100 откликов за раз, для получения подробной информации. В ответе присутствует `vacancy_id `- по которому можно определить вакансию, по которой был совершен отклик.

### Работа с откликами по расписанию (cron)
##### Алгоритм интеграции:
1. [Авторизация](#ApiDocumentationBlock), получить токен временного доступа.
2. [Получить список ваших вакансий](../item/documentation#operation/getItemsInfo), для фильтрации необходимо передать `category=111`, где 111 - это категория вакансии. Сохранить id из ответа.
3. [Получить отклики по вакансиям](#operation/applicationsGetByIds), укажите следующие параметры и сохраните applies.id из ответа:
   * `vacancyIds` список вакансий который получили на 2 этапе. 
   * `updatedAtFrom=<YYYY-MM-DD>` вернется список откликов с этой даты по настоящее время.
   * `cursor` идентификатор последнего отклика из предыдущего запроса.
   * `is_viewed ` вернется список откликов по статусу: `true `- просмотрен, `false `- не просмотрен.
4. Работа с `cursor`, необходим для пагинации, ограничение на количество откликов в одном ответе 100 штук.
   * При первом запросе cursor не передаем.
   * При втором и последующем запросе cursor необходимо указывать равным id, где id - это последний элемент в массиве applies. 
5. [Получить информацию по откликам](#operation/applicationsGetByIds), передайте идентификаторы откликов в качестве параметра ids, не больше 100 откликов за раз, для получения подробной информации.
6. Настройте расписание запуска алгоритма по вашим требованиям, но не чаще чем один раз в 5 минут. Для более оперативного получения откликов используйте webHook.

### Работа с содержимым откликов

[Получение информации по отклику](#operation/applicationsGetByIds), уточнение по некоторым полям из ответа:
  * `applicant.data `- Информация о кандидате которую он заполнил о себе.
  * `contacts.chat `- Идентификатор чата, [просмотр содержимого чата](../messenger/documentation#operation/chatRead), если способ связи только звонок, то chat будет пустым
  * `contacts.phones `- Номер телефона соискателя в формате `72002000014`. Кандидат может отказаться от передачи номера телефона.
  * `employee_id `- Идентификатор сотрудника, если вакансия была опубликована от "имени" сотрудника в иерархии аккаунтов.
  * `type `- Информация о том, каким образом был совершен отклик: `by_chat `- кандидат откликнулся, `by_call `- кандидат просмотрел номер телефона из вакансии.
  * `vacancy_id `- Идентификатор вакансии, [подробную информацию по вакансии](#operation/vacancyGetItem).
  * `is_viewed `- Информация о том, просмотрен ли отклик: `true `- просмотрен, `false `- не просмотрен.

### Отклик и чат-бот
Если соискатель откликнулся на вакансию в которой есть чат-бот, то в список откликов и в webhook он попадет только после его прохождения или же по истечению времени прохождения опроса (1 час).

Результат работы чат-бота можно найти в методе [Получение информации по отклику](#operation/applicationsGetByIds) в разделе `prevalidation`.
* `prevalidation.status `- Описание статусов: `not_complete `- кандидат начал проходить опрос, но не дошел до конца, `success `- кандидат прошел опрос, `failed `- кандидат прошел чат-бота до конца, но не подошел по требованиям.
* `prevalidation.summary `- Результат опроса: `label `- текстовое описание вопроса к кандидату, `value `- ответ кандидата. Пример  `{"label": "Возраст","value": "20"}`.

### Данные из личного кабинета соискателя в откликах
Если соискатель заполнял данные о себе на Авито Работе в личном кабинете или при откликах, то собранную информацию и 
результаты её валидации по критериям вакансии можно посмотреть в методе [Получение информации по отклику](#operation/applicationsGetByIds)
в разделе `enriched_properties`:
 * `enriched_properties.status `- текущий статус сбора информации о кандидате
 * `enriched_properties.citizenship `- гражданство кандидата
 * `enriched_properties.experience `- опыт работы по профессии кандидата
 * `enriched_properties.phone `- номер телефона кандидата
 * `enriched_properties.gender `- пол кандидата
 * `enriched_properties.full_name `- ФИО кандидата

## Публикация вакансии
Для публикации вакансий через api, необходимо приобрести [тариф](https://www.avito.ru/tariff/cpa/configure?verticalId=2) на размещение.

### Асинхронный способ подачи вакансии
Рекомендуем использовать этот способ.
 * Минимальное время ожидания ответа от метода публикации вакансии.
 * Быстрая проверка статуса публикации вакансии.
 * Развернутые ошибки, которые были допущены при заполнении вакансии.
 * Вакансия опубликуется автоматически, даже при наличии внутренних сбоев сайта Avito.

1. [Опубликуйте вакансию](#operation/vacancyCreateV2) и сохраните `id` из ответа - *этот id не является идентификатором вакансии*. 
2. Уточнения по некоторым полям:
   * `image_url `- ссылка на логотип вакансии. Файл должен располагаться на общедоступном ресурсе, без каких либо ограничений для скачивания, так же иметь максимальный размер 24 мб в формате: jpeg, png, gif.
   * `billing_type `: `package `- размещение объявления осуществляется только при наличии подходящего пакета размещения, `packageOrSingle `- при наличии подходящего пакета оплата размещения объявления произойдет с него, если нет подходящего пакета, но достаточно денег на кошельке Авито, то произойдет разовое размещение.
3. [Проверяем статус вакансии](#operation/vacancyGetStatuses), передаем id который вернулся при публикации. 
   * `vacancy.id `- идентификатором вакансии.
   * `vacancy.url `- ссылка на вакансию на сайте Avito.
   * `vacancy.status `: `new `- вакансия сохранена и готова к публикации, `process `- вакансия в процессе публикации, `success `- вакансия опубликована, `error `- во время публикации вакансии произошла ошибка, детали ошибки смотрите в `last_action.error.message`.
   * `vacancy.moderation_status`: значение отсутствует - процесс модерации вакансии еще не начался, `in_progress`- вакансия в процессе модерации, `allowed`- вакансия разрешена на модерации, `blocked`- вакансия заблокирована на модерации, `rejected`- вакансия отклонена на модерации.
4. Сохраните связку `id` публикации и `vacancy.id`, для удобства обновления вакансии.
5. [Обновление вакансии](#operation/vacancyUpdateV2), метод так же возвращает id публикации, после обновления публикация переходит в статус `new`.
6. [Просмотр деталей вакансии](#operation/vacancyGetItem), передайте `fields` и `params`, что бы получать только необходимую информацию. По умолчанию отображаются все поля.
7. [Получение статистики по вакансии](../item/documentation#operation/itemStatsShallow), получаем статистику количества просмотров, откликов и добавление в избранное за выбранный период времени.
8. [Получение информации о стоимости услуг продвижения](../item/documentation#operation/vasPrices), передаем `vacancy.id` и получаем информацию о тратах на продвижение вакансии, без разбивки по датам.

### Синхронный способ подачи вакансии (deprecated)
1. [Опубликуйте вакансию](#operation/vacancyCreate), контракт такой же как и в [асинхронном способе](#асинхронный-способ), но при публикации возвращается идентификатор вакансии и ссылка на сайт Avito.
2. Публикация таким способом занимает продолжительное время 3-5с, при высокой нагрузки время может увеличится до 1м.
3. [Обновление вакансии](#operation/vacancyUpdate), так же занимает продолжительное время как и создание.
4. [Просмотр деталей вакансии](#operation/vacancyGetItem), передайте `fields` и `params`, что бы получать только необходимую информацию. По умолчанию отображаются все поля.
5. [Получение статистики по вакансии](../item/documentation#operation/itemStatsShallow), получаем статистику количества просмотров, откликов и добавление в избранное за выбранный период времени.
6. [Получение информации о стоимости услуг продвижения](../item/documentation#operation/vasPrices), передаем `vacancy.id` и получаем информацию о тратах на продвижение вакансии, без разбивки по датам.

## Иерархия аккаунтов
В Авито существует две сущности: Компания и Сотрудник.
* Если сотрудник авторизовался в API под своим аккаунтом, то он работает с методами как будто он не связан с компанией.
* Для того чтобы в API сотруднику был связан с компанией необходимо передать заголовок `X-Is-Employee:true`.
* При передаче заголовка `X-Is-Employee:true` на сотрудника распространяется тариф компании.
* Не все методы поддерживают данный заголовок, смотрите их описание.
* [Полезные методы работы с иерархией аккаунтов](../accounts-hierarchy/documentation)
  * Получение информации о статусе пользователя в Иерархии Аккаунтов - можно узнать под каким пользователем сейчас идет авторизаци и кем он является isCompany - головной аккаует, isChief - руководитель и isEmployee - сотрудник.
  * Получение списка сотрудников иерархии - если вы компания или руководитель, то можете получить список своих подчиненных.
  * Прикрепление сотрудника иерархии к объявлениям, перезакрепление объявлений между сотрудниками иерархии - при смене сотрудника прикрепляется и список откликов к этой вакансии.
  * Получение списка телефонов компании - только активные номера.
  * Получение списка объявлений по сотруднику - сотрудник компании может получить список своих объявлений если передаст в качестве параметра `employeeId` самого себя.

### Публикация вакансии в иерархии
* Существует два способа опубликовать вакансию с привязкой к сотруднику:
  1. Через аккаунт компании - при публикации вакансии передать параметр `hierarchy.employee_id`
  2. Через аккаунт сотрудника - передайте заголовок `X-Is-Employee: true`. В этом случае передавать поле `hierarchy.employee_id` не требуется.
* Оплата публикации 
  * Оплата за размещение вакансии производится только из пакета сотрудника `billing_type: package`.
  * Управление лимитами сотрудников доступно на странице [Сотрудники](https://www.avito.ru/employees).
