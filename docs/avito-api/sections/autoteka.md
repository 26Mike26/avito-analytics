# Автотека (`autoteka`)

## Общая информация

Для того чтобы воспользоваться данным функционалом, вам необходимо стать партнером Автотеки.

Стать официальным партнером и получить доступ к данным методам можно,
подав заявку через [Обратную связь с Autoteka](https://autoteka.ru/feedback).

В качестве темы запроса нужно указать **"Сотрудничество"**.

В рамках публичного API существует ряд сценариев использования:
* Отчет по авто
* Сервис тизеров
* Сервис ТТХ
* Оценка стоимости авто по ТТХ
* Сервис скоринга рисков

Для использования каждого из сценариев необходимо использовать отдельную учетную запись Автотеки.

## Обратная совместимость

Количество и набор данных в отчете со временем увеличивается, мы следим за обратной совместимостью и не вносим структурные изменения в ранее созданные блоки.
Объекты могут только расширяться новыми полями.
Структурные изменения объектов или удаления полей (нарушение обратной совместимости) будут доступны через новую версию (префикс v, текущая версия v1).

## Обработка исключительных ситуаций

Если данных нет по каким-либо причинам, то в теле ответа будет передано значение <b>null</b>, если поле указано как <b>nullable</b>.
Поле может отсутствовать, если оно не перечислено в блоке <b>required</b>.

## Технические особенности
Следует учесть, что конечные ip-адреса могут меняться. Если вы используете белый список ip, то
запросите, пожалуйста, у нас актуальный диапазон адресов.

## Примеры справочных значений
Примеры данных, которые могут быть в полях json-отчёта, мы указываем в поле спецификации <b>enum</b>.
<b>Важно:</b> это не исчерпывающий список значений, при интеграции следует учитывать, что могут приходить иные значения.
Данные приведены для примера, чтобы можно было сделать вывод о том, какого вида информация приходит в поле.

Пути API могут перемещаться, поэтому следует корректно обратабывать 301, 302-редиректы.

## Отчёт по авто
### Алгоритм получения отчёта

1. [Получить](#operation/getAccessToken) токен временного доступа.
2. Отправить [запрос](#operation/postPreviewByVin) на создание превью по VIN или [запрос](#operation/postPreviewByRegNumber) на создание превью по государственному номеру.
3. Отправить [запрос](#operation/getPreview) на получение превью по ID (обычно превью формируется быстро, поэтому можно запрашивать статус раз в 2-5 секунд).
4. Проверить статус:
    - Если <b>processing</b>, то повторить шаг 3.
    - Если <b>notFound</b>, то это значит, что по данному VIN нельзя сформировать отчёт.
    - Если <b>success</b>, то перейти к п.5.
5. Отправить [запрос](#operation/postReport) на создание отчёта по превью.
6. [Проверить](#operation/getReport) статус по ID отчета:
    - Если <b>processing</b>, то повторить шаг 5.
    - Если <b>success</b>, то в объекте <b>report.result.data</b> будут данные отчета.
7. Если значение поля <b>result.report.data.serviceInfo.reportCompleteStatus.status</b> = <b>incomplete</b> то это значит, что не все источники данных были доступны на момент формирования отчета. Нужно повторно запросить статус отчета через некторое время (п. 6).

## Сервис «Сигнал»

### Получение списка лидов
Фид нужно читать последовательно. Для этого:
1. Выполнить запрос на [чтение фида](#operation/getLeads) без параметра lastId
2. Обработать массив полученных лидов, запомнить последний id
3. Выполнить запрос на чтение фида с параметром lastId, заполненным значением из п.2


## Оффлайн мониторинг

### Общая информация
Регистрационное действие — строчка в ПТС, это может быть смена владельца, снятие с учёта и т. д.
При интеграции с API следует учесть, что в процессе разработки передаваемые объекты могут расширяться новыми полями. Структурные изменения объектов или удаления полей (нарушение обратной совместимости) будут доступны через новую версию (префикс v, текущая версия v1).

### Схема взаимодействия
Взаимодействие с API мониторинга регистрационных действий состоит из двух этапов:
1. Загрузка списка VIN-номеров
2. Получение информации по рег. действиям.

Список VIN-номеров можно как пополнять, так и очищать полностью с последующей актуализацией.
При добавлении VIN-номера по нему начинает подтягиваться информация по совершенным рег. действиям с момента добавления в сервис.

Получения изменений рекомендуется делать ежедневно.
Так как набор изменений может быть объемным, API предусматривает постраничную выдачу информации с помощью курсоров.
Чтобы продолжить вычитывать свежие изменения, следует выполнять запросы с учетом последней позиции курсора.

- [Очистка VIN-номеров](#operation/monitoringBucketDelete)
- [Выборочное удаление VIN-номеров](#operation/monitoringBucketRemove)
- [Получение регистрационных действий](#operation/monitoringGetRegActions)

## Сервис ТТХ
### Общая информация
Набор методов для получения нормализованных технических характеристик по государственному регистрационному знаку или идентификатору авто.

### Схема взаимодействия
Взаимодействие с API сервиса ТТХ состоит из двух этапов:
1. Запрос получения спецификации
2. Получение спецификации

Первым запросом необходимо запросить спецификацию по идентификатору авто при помощи метода [Запрос получения спецификации по идентификатору авто](#operation/specificationByVehicleId) или по госномеру при помощи метода [Запрос получения спецификации по госномеру](#operation/specificationByPlateNumber).
В ответе будет предоставлен `specificationId`. Для использовании метода [получения спецификации по госномеру](#operation/specificationByPlateNumber) необходимо обратиться к менеджеру Автотеки.

Для получения спецификации с полученным `specificationId` необходимо обратиться к методу [Получение спецификации](#operation/specificationGetById), который может в ответе вернуть поле `Status`, если оно имеет значение `processing`, значит необходимо повторить запрос позже.

### Методы для работы
- [Запрос получения спецификации по идентификатору авто](#operation/specificationByVehicleId)
- [Запрос получения спецификации по госномеру](#operation/specificationByPlateNumber)
- [Получение спецификации](#operation/specificationGetById)

## Сервис оценки стоимости автомобиля

Набор методов для получения оценки автомобиля. Может выдавать оценку как по техническим характеристикам, так и по государственному регистрационному знаку или идентификатору авто.

### Схема взаимодействия

Взаимодействие с API сервиса оценки автомобиля состоит из двух этапов:
1. Запрос спецификации авто.
2. Получение оценки по спецификации автомобиля.

Спецификацию авто можно получить двумя способами:
1. Через сервис ТТХ при помощи методов [Запрос получения спецификации по идентификатору авто](#operation/specificationByVehicleId) и [Запрос получения спецификации по госномеру](#operation/specificationByPlateNumber). При использовании сервиса ТТХ потребуются две учетные записи, одна для получения параметров авто, а вторая для оценки.
2. При помощи каталога параметров. Для этого требуется воспользоваться методом [получения актуальных параметров каталога](#operation/catalogsResolve). Данный метод необходимо перезапрашивать несколько раз для сбора всех необходимых параметров для оценки. Необходимые поля можно посмотреть в обязательных параметрах метода [Получение оценки по спецификации автомобиля](#operation/valuationBySpecification).

После получения спецификации необходимо отправить ее в метод [Получение оценки по спецификации автомобиля](#operation/valuationBySpecification), который в ответе вернет оценку автомобиля по данной спецификации.
### Методы для работы
- [Получение актуальных параметров каталога](#operation/catalogsResolve)
- [Получение оценки по спецификации автомобиля](#operation/valuationBySpecification)
