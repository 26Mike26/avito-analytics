# Подключение Supabase + публичный деплой — пошаговая инструкция

Это превратит локальный прототип в полноценный мульти-пользовательский SaaS:
один URL, на который можно зайти с любого устройства, у каждого пользователя
свои аккаунты и данные, журнал действий и т.д.

Всё делается один раз и занимает 15–25 минут. Дальше — просто `git push` и
Vercel сам пересоберёт.

> **Почему вы делаете это сами, а не я.** Из своей рабочей среды я не могу
> заходить на supabase.com / github.com / vercel.com — они вне белого списка
> сетевых соединений, а принимать ваши пароли/токены в чате запрещено
> правилами безопасности. Поэтому я максимально автоматизировал всё, что
> можно автоматизировать без чужих учёток: scripts/setup.sh для локала,
> Deploy-кнопка для Vercel, готовая SQL-миграция. Каждый клик ниже —
> либо в вашем браузере, либо в вашем терминале.

---

## Шаг 0. Что у вас должно быть

- Аккаунт на [github.com](https://github.com/) (нужен для бесплатного Vercel).
- Браузер.
- Терминал (Mac: «Терминал», Windows: PowerShell).
- Установленный Node.js 18+ ([nodejs.org](https://nodejs.org/)).

---

## Шаг 1. Создайте проект Supabase

1. Откройте [supabase.com](https://supabase.com/) → **Start your project** → войдите через GitHub.
2. Нажмите **New Project**.
3. Заполните:
   - **Name**: `avito-analytics`
   - **Database password**: придумайте, **запишите** в надёжное место.
   - **Region**: ближайший к пользователям, например `Frankfurt (eu-central-1)`.
   - **Pricing plan**: Free.
4. Нажмите **Create new project**. Подождите 1–2 минуты, пока проект создастся.

---

## Шаг 2. Примените SQL-схему

1. В созданном проекте слева в меню выберите **SQL Editor** → **New query**.
2. Откройте файл `supabase/schema.sql` из этого репозитория, скопируйте всё его содержимое.
3. Вставьте в редактор и нажмите **Run** (или `Cmd/Ctrl+Enter`).
4. Внизу должно появиться `Success. No rows returned`. Это значит таблицы и
   политики безопасности (RLS) созданы.

> Безопасно прогонять повторно — все запросы используют `IF NOT EXISTS` /
> `DROP POLICY IF EXISTS`.

---

## Шаг 3. Скопируйте URL и публичный ключ

1. В Supabase → **Settings** (шестерёнка слева внизу) → **API**.
2. Скопируйте два значения:
   - **Project URL** (например, `https://abcdefgh.supabase.co`)
   - **anon public** (длинный ключ, начинается с `eyJ...`)

> ⚠️ **service_role** ключ копировать НЕ нужно — он даёт админ-доступ к БД и
> в браузере его быть не должно.

---

## Шаг 4. Положите ключи в `.env`

В корне проекта (там, где `package.json`):

```bash
cp .env.example .env
```

Откройте `.env` и заполните:

```
VITE_SUPABASE_URL=https://abcdefgh.supabase.co
VITE_SUPABASE_ANON_KEY=eyJ.....
```

Сохраните файл. **Не коммитьте `.env`** — он в `.gitignore`.

---

## Шаг 5. Установите зависимость и протестируйте локально

```bash
npm install
npm run dev
```

Откроется `http://localhost:5173`. Должна появиться страница «Вход».

Тест:
1. Нажмите **Зарегистрироваться**, создайте аккаунт.
2. Откройте Supabase → **Authentication → Users** — пользователь должен появиться.
3. Откройте **Table Editor → accounts** — должен появиться один демо-аккаунт.
4. Зайдите в Settings, импортируйте свой CSV → проверьте, что строки появились в таблицах
   `items` и `metrics`.
5. Откройте сайт в другом браузере (или режиме инкогнито), войдите тем же логином — должны
   появиться те же данные. Это и есть тот самый «один URL для всех».

> Если регистрация требует подтверждения email — Supabase → **Authentication → Providers
> → Email** → выключите «Confirm email» (для прототипа). На бой включите обратно.

## Шаг 6. Положите проект в GitHub

```bash
cd avito-analytics
git init
git add .
git commit -m "init"
git branch -M main
```

Создайте пустой репозиторий на github.com (например, `avito-analytics`), потом:

```bash
git remote add origin https://github.com/<ваш-логин>/avito-analytics.git
git push -u origin main
```

---

## Шаг 7. Задеплойте на Vercel

1. Откройте [vercel.com](https://vercel.com/), войдите через GitHub.
2. **Add New → Project** → выберите ваш репозиторий `avito-analytics`.
3. **Framework Preset**: Vite (определится автоматически).
4. Раскройте **Environment Variables** и добавьте две переменные:
   - `VITE_SUPABASE_URL` — то же, что в `.env`.
   - `VITE_SUPABASE_ANON_KEY` — то же, что в `.env`.
5. Нажмите **Deploy**. Через 1–2 минуты получите ссылку вида
   `https://avito-analytics-xxxx.vercel.app`. Это и есть ваш SaaS.

> Каждый `git push` в `main` автоматически пересобирает и обновляет сайт.

---

## Шаг 8. (опционально) Подключите свой домен

1. В Vercel → проект → **Settings → Domains** → **Add** → введите свой домен.
2. Vercel покажет, какие DNS-записи нужно добавить у вашего регистратора домена.
3. Через 5–60 минут домен подцепится. SSL Vercel выпустит сам.

## Что дальше

Готово: у вас публичный URL, регистрация/вход через Supabase, мульти-аккаунт,
журнал действий — всё с синхронизацией между устройствами.

Что осталось для боевого продукта (по приоритету):

1. **Шифрование секретов Avito.** В таблице `accounts` сейчас `integration_client_secret`
   и `integration_access_token` хранятся открыто. Включите Postgres-расширение `pgsodium`
   и используйте `crypto_aead_det_encrypt(...)` или вообще не возвращайте эти поля на
   фронтенд (читать их должна только серверная Edge Function).
2. **Avito API через серверный прокси.** Папка `backend/` — это заготовка Express-сервера.
   Деплой на Render/Railway/Fly.io, в `.env` положите свои Client ID/Secret. Worker должен
   ходить в Supabase, читать аккаунты в режиме `api`, обновлять данные.
3. **Cron-синхронизация.** Включите расширение `pg_cron` в Supabase или внешний крон.
4. **Подтверждение email + восстановление пароля.** Включите в Supabase → Authentication.
5. **Юр. часть.** Политика конфиденциальности, согласие на обработку, при необходимости
   регистрация ИП/ООО как оператора персданных.

## Если что-то пошло не так

- **«Failed to fetch» при регистрации.** Проверьте, что `VITE_SUPABASE_URL` начинается с
  `https://` и что вы перезапустили `npm run dev` после правки `.env`.
- **«Permission denied for table»** в Supabase logs. Значит RLS-политики не применились —
  заново прогоните `supabase/schema.sql` целиком.
- **На Vercel сайт пустой / 404 при перезагрузке внутренних страниц.** Проверьте, что в
  репозитории есть `vercel.json` (он там есть) и что в Vercel Settings проставлены ENV.
- **На сайте всё работает в локальном режиме (без Supabase).** Значит ENV не подгрузились
  при сборке. В Vercel: Settings → Environment Variables → Save → потом Deployments →
  последний деплой → **Redeploy**.
